    IBM Bob 活用ガイド

    このガイドでは、チュートリアルで学んだ内容を振り返り、IBM Bobをより効果的に活用するための実践テクニックとリファレンス情報を提供します。

    Part 1: チュートリアルの総括

    学習内容の振り返り

    このチュートリアルでは、IBM Bobを使った実践的な開発フローを体験しました：

    LAB1: ゼロからのサイト構築 - 要件定義から実装までの一連の流れ - Orchestrateモードによる自動タスク分解 - HTML/CSS/JavaScriptによるFAQサイトの作成

    LAB2: 既存ファイルの活用と機能拡張 - 既存ファイル（qa.md）の読み込みと活用 - 翻訳機能の実装 - 画像の挿入とデザイン調整

    LAB3: コード品質の向上 - Advancedモードへの切り替え - /reviewコマンドによるコードレビュー - 指摘事項の優先順位付けと対応

    達成度チェック

    以下の項目を確認して、学習の定着度を確認しましょう：

    • [ ] IBM Bobの基本的な画面構成を理解している
    • [ ] 各モード（Orchestrate、Plan、Code、Ask、Advanced）の違いを説明できる
    • [ ] 効果的なプロンプトを書くことができる
    • [ ] ファイルの読み込みと編集の承認フローを理解している
    • [ ] 自動承認機能の使い分けができる
    • [ ] レビュー機能を使ってコード品質を確認できる
    • [ ] TODOリストを活用してタスクを管理できる

    重要なポイント

    1. 段階的なアプローチ - 大きなタスクは小さなステップに分解 - 各ステップで結果を確認しながら進める - 問題が発生したら早めに軌道修正

    2. 適切なモード選択 - タスクの性質に応じてモードを使い分ける - 複雑なタスクはOrchestrateモードから開始 - 単純な質問はAskモードで効率的に

    3. コミュニケーション - 具体的で明確な指示を心がける - 必要な情報は事前に整理して提供 - 期待する結果を明確に伝える

    Part 2: 実践テクニック

    1. 効果的なプロンプト作成

    プロンプトの質が、IBM Bobの出力品質を大きく左右します。

    ✅ 良い例

    以下の機能を持つFAQサイトを作成してください：

【機能要件】
- 検索窓でキーワード検索
- カテゴリ絞り込み
- アコーディオン形式で質問と回答を表示
- 日本語翻訳済みのFAQデータを使用

【技術要件】
- HTML/CSS/JavaScriptのみで実装
- ローカル環境で動作
- レスポンシブデザイン

【データ】
- qa.mdファイルを参照

    良い点: - 機能要件が明確 - 技術的な制約が指定されている - 参照すべきファイルが明示されている - 構造化されていて読みやすい

    ❌ 悪い例

    FAQサイトを作って

    問題点: - 要件が曖昧 - 技術スタックが不明 - データソースが不明 - 期待する機能が不明確

    プロンプト作成のコツ

    1. 5W1Hを意識する
    2. What: 何を作るのか
    3. Why: なぜ必要なのか
    4. Who: 誰が使うのか
    5. When: いつまでに必要か
    6. Where: どこで動作するのか

    7. How: どのように実装するのか

    8. 段階的に詳細化する

    9. 最初は大まかな要件から
    10. Bobの質問に答えながら詳細化

    11. 必要に応じて追加情報を提供

    12. 例を示す

    13. 期待する出力の例を提示
    14. 参考となるサイトやコードを共有
    15. 具体的なイメージを伝える

    2. モード活用戦略

    各モードの特性を理解し、適切に使い分けることで効率が向上します。

    モード 使用場面 特徴 使用例
    Orchestrate 複雑なプロジェクト全体 複数モードを自動調整 新規アプリケーション開発
    Plan 設計・要件定義 仕様書生成 アーキテクチャ設計
    Code コード作成・修正 実装に特化 バグ修正、機能追加
    Ask 質問・相談 情報提供 技術的な質問、調査
    Advanced 複雑なタスク 拡張機能利用 コードレビュー、リファクタリング

    モード選択のフローチャート

    タスクの性質は？
├─ 質問・情報収集 → Ask
├─ 設計・計画 → Plan
├─ コード実装
│  ├─ 単純な修正 → Code
│  └─ 複雑な実装 → Orchestrate
└─ レビュー・分析 → Advanced

    3. 自動化のコツ

    自動承認機能を適切に使うことで、作業効率が大幅に向上します。

    初心者の場合

    推奨設定: - 自動承認はオフ - 各ステップで内容を確認 - Bobの動作パターンを学習

    メリット: - 安全性が高い - 学習効果が高い - 予期しない変更を防げる

    慣れてきたら

    推奨設定: - ファイル書き込みを自動承認 - 定型的な操作を自動化 - 重要な操作は手動確認

    メリット: - 作業効率が向上 - 反復作業が削減 - フローが中断されない

    自動承認の設定タイミング

    プロジェクトの段階に応じて調整：

【初期段階】
- 自動承認: オフ
- 理由: 要件が固まっていない

【開発段階】
- 自動承認: 部分的にオン
- 理由: 定型作業が増える

【保守段階】
- 自動承認: 選択的にオン
- 理由: 影響範囲が限定的

    4. レビュー機能の活用法

    定期的なコードレビューで品質を維持します。

    基本的な使い方

    /review

    このコマンドで、プロジェクト全体のコードレビューが実行されます。

    レビューのタイミング

    1. 機能実装後
    2. 新機能追加時
    3. 大きな変更後

    4. マージ前

    5. 定期的なレビュー

    6. 週次レビュー
    7. スプリント終了時

    8. リリース前

    9. 問題発生時

    10. バグ発見時
    11. パフォーマンス低下時
    12. セキュリティ懸念時

    優先度に基づく対応

    レビュー結果は優先度順に対応します：

    1. セキュリティ問題（最優先）
    2. 即座に対応
    3. 他の作業を中断してでも修正

    4. 例: SQLインジェクション、XSS脆弱性

    5. パフォーマンス問題

    6. 影響度を評価
    7. 計画的に対応

    8. 例: N+1クエリ、メモリリーク

    9. 保守性の問題

    10. リファクタリング計画に組み込む
    11. 段階的に改善
    12. 例: コードの重複、命名規則

    レビュー結果の活用

    レビュー → 分類 → 優先順位付け → 対応計画 → 実装 → 再レビュー

    Part 3: リファレンス

    よくある質問（FAQ）

    基本的な質問

    Q: IBM Bobは無料で使えますか？
    A: 無料プランと有料プランがあります。基本機能は無料プランで利用可能です。詳細は公式サイトをご確認ください。

    Q: どのプログラミング言語に対応していますか？
    A: JavaScript、Python、Java、C++、Go、Rust、TypeScript、PHP、Ruby、Swiftなど、主要な言語に対応しています。

    Q: オフラインでも使えますか？
    A: 基本的にはインターネット接続が必要ですが、一部のキャッシュ機能はオフラインでも利用可能です。

    Q: 生成されたコードの著作権は？
    A: 生成されたコードの著作権はユーザーに帰属します。ただし、利用規約をご確認ください。

    Q: セキュリティは大丈夫ですか？
    A: IBM Cloudのセキュリティ基準に準拠しており、コードは暗号化されて送信されます。機密情報の取り扱いには注意してください。

    技術的な質問

    Q: 既存のプロジェクトに途中から導入できますか？
    A: はい、可能です。プロジェクトフォルダを開くだけで、既存のコードを認識して作業できます。

    Q: チーム開発で使えますか？
    A: はい、各メンバーが個別にIBM Bobを使用できます。ただし、コードの競合には注意が必要です。

    Q: 生成されたコードの品質は？
    A: 一般的に高品質ですが、必ずレビューと動作確認を行ってください。特にセキュリティ面は重要です。

    Q: カスタマイズはできますか？
    A: 設定ファイル（.bobignore等）でカスタマイズ可能です。詳細は公式ドキュメントを参照してください。

    トラブルシューティング

    接続エラー

    症状: - IBM Bobが応答しない - タイムアウトエラーが発生

    対処法: 1. Wi-Fi接続を確認 2. プロキシ設定を確認 3. ファイアウォール設定を確認 4. IBM Bobを再起動 5. ネットワーク管理者に相談

    拡張機能が動作しない

    症状: - コマンドが実行されない - ファイルが認識されない

    対処法: 1. VS Codeを再起動 2. フォルダーを開き直す 3. 設定ファイル（settings.json）を確認 4. 拡張機能を再インストール 5. VS Codeのバージョンを確認

    パフォーマンスが遅い

    症状: - 応答が遅い - 処理に時間がかかる

    対処法: 1. ネットワーク接続速度を確認 2. 不要なファイルやタブを閉じる 3. より小さいモデルを使用 4. .bobignoreで不要なファイルを除外 5. プロジェクトサイズを確認

    ファイルが認識されない

    症状: - 作成したファイルが表示されない - 変更が反映されない

    対処法: 1. エクスプローラーを更新（F5） 2. フォルダーを開き直す 3. ファイルパスを確認 4. .bobignoreの設定を確認 5. ファイルの保存を確認

    用語集

    用語 説明
    Orchestrate 複数のモードを自動的に切り替えながらタスクを実行するモード
    Plan 要件定義や設計書の作成に特化したモード
    Code コードの作成・編集に特化したモード
    Ask 質問や情報収集に特化したモード
    Advanced 高度な機能（MCP等）を利用できるモード
    自動承認 Bobの操作を自動的に承認する機能
    TODOリスト タスクの進捗を管理するチェックリスト
    .bobignore Bobに読み取らせないファイルを指定する設定ファイル
    MCP Model Context Protocol - 外部ツールとの連携機能
