# LAB1 - 初期開発
![LAB1 Header](assets/images/03_lab1/lab1_header.png)

このラボでは、IBM Bobを使って、手元で開けるシンプルなFAQサイトを作ることを目指します。


### ゴール
機能要件に従ったFAQサイトをノーコードで作成し、挙動の確認まで行う。

## 要件

### サイトの動き方
- すべて1つのページの中で動作する

### 実装する機能
- 検索窓でキーワード検索
- カテゴリ絞り込み
- FAQの一覧表示（日本語翻訳済み）
- 質問をクリックすると答えが開く（アコーディオン）

### 技術スタック
- ローカル環境で動く（ブラウザで直接開く仕様）
- HTML / CSS / JavaScript だけで作成

### サイトの完成例

![Site Example](assets/images/03_lab1/site_example.png)

## Promptの入力

### 1. Promptのコピー

PROMPT.mdを開き、中の文章をコピーします。

**ユーザーのアクション:**
- PROMPT.mdファイルを開く
- 内容をコピー

### 2. Promptのコピー & 実行

PROMPT.mdをチャットに貼り付け、**Orchestratorモード**を選択し、右下の実行ボタンを押します。

![Prompt Paste](assets/images/03_lab1/prompt_paste.png)

## サブタスクの作成

### サブタスクの作成

promptの内容をもとにBobがサブタスクの計画を作成しますので、承認します。

**Bobのアクション:**
- Promptを解析
- サブタスクを生成
- 承認を求める

**ユーザーのアクション:**
- サブタスクの内容を確認
- 「承認」をクリック

![Subtask Approval](assets/images/03_lab1/subtask_approval.png)


## 仕様書の作成

### 1. 読み取りの許可

仕様書を作成するためにBobはファイルの読み取り許可を求めますので、「承認」をクリックします。
![alt text](assets/images/image-7.png)
**Bobのアクション:**
- qa.mdファイルの読み取り要求
- FAQデータの解析

**ユーザーのアクション:**
- 「承認」をクリック

### 2. 要件の指示

設計フェーズでは、Bobが細かな要件を尋ねてくれるので、初期開発の要件を参考にクリックしていきます。

**注意:** 生成AIの生成結果によって尋ねられる要件の内容は変わることがあります。

**FAQサイトのイメージについての要件定義の例:**
![Requirements Definition](assets/images/03_lab1/requirements_definition.png)

### 3. 設計用タスクの承認

要件の定義後、Bobがユーザーの指示した要件に応じて、設計用のTODOリストを生成します。その内容に問題がなければ、「承認」を押します。

承認後は生成されたTODOリストに従い、段階的にタスクが実行されていきます。

**注意:** 生成AIの生成結果によってTODOリストの内容は変わることがあります。
![Design TODO Approval](assets/images/03_lab1/design_todo_approval.png)

### 4. 仕様書の生成

TODOリストを承認するとBobはFAQサイトの仕様書を生成します。

**生成される内容:**
- レイアウト構成
- 技術スタック
- 機能仕様
- データ構造

**ユーザーのアクション:**
- 仕様書ファイルがディレクトリに追加されることを確認

![Specification Generation](assets/images/03_lab1/spec_generation.png)

### 5. 仕様書の確認

仕様書の生成には時間がかかるので、その間に生成されている仕様書の内容を確認します。必要な要件が仕様書に反映されているかを確認してください。

**確認ポイント:**
- レイアウト構成
- 技術スタック
- 機能要件
- データ構造

![alt text](assets/images/image-8.png)

### 6. 仕様書の保存

作成された仕様書を保存するために許可を求められるので、「保存」をクリックします。
![alt text](assets/images/image-9.png)

### 7. 仕様書作成タスクの完了

仕様書の生成後に、TODOリストの更新をBobが行うので「承認」を押します。
![alt text](assets/images/image-10.png)

## 設計フェーズの完了

### 1. 設計フェーズの完了

設計フェーズが終わると、左のようなサマリーをBobが出力します。


**サマリーの内容:**
- 完了したタスク
- 生成されたファイル
- 次のステップ
![alt text](assets/images/image-14.png)

### 2. サブタスクの完了

サマリーを下にスクロールすると、サブタスクの完了指示があるので、「サブタスクを完了して戻る」ボタンをクリックします。
![alt text](assets/images/image-12.png)

## 実装計画の作成

### 1. 設計タスクの総括

先ほどまでのサブタスクの結果が親タスクに返されますので承認をクリックします。

![alt text](assets/images/image-11.png)

### 2. ファイルの読み取りの許可

セッションが切り替わりファイルの読み取りの許可を求められるので「すべて承認」をクリックします。

![alt text](assets/images/image-15.png)

### 3. 実装計画の方針定義

下にスクロールすると、実装計画の方針についてBobがユーザに尋ねてきます。

**方針の例:**
- 段階的な実装
- 優先機能の選択
- テスト戦略

![alt text](assets/images/image-16.png)

### 4. 実装計画の承認

方針決定後に、実装計画をBobが提示するため、問題がなければ「承認」をクリックします。

![alt text](assets/images/image-17.png)

## コードの生成

### 1. ディレクトリの作成

実装するためのディレクトリ構造のためにBobがコマンドを提案するので、「コマンド」ボタンをクリックします。

![alt text](assets/images/image-30.png)


### 2. コードの生成

ディレクトリ構造作成後、Bobはコードファイルを生成します。

**生成されるファイル:**
- index.html
- styles.css
- script.js
- その他必要なファイル

![alt text](assets/images/image-19.png)

### 3. コードの書き込みの承認

ファイルが生成されるたびに、保存の承認を依頼されるため、ユーザーは、「承認」をクリックします。

![alt text](assets/images/image-20.png)

この時、適宜、**自動承認モード**を活用し、書き込み承認の自動設定を行います。

![alt text](assets/images/image-21.png)

### 4. 生成されたコードの確認

生成されたコードを確認します。

**確認ポイント:**
- HTML構造
- CSSスタイル
- JavaScript機能
- 仕様書との整合性

![alt text](assets/images/image-22.png)

### 5. タスクの完了

コーディングが完了したら、「承認」をクリックし、コード生成タスクを完了します。

![alt text](assets/images/image-23.png)

### 6. README.mdの作成

コード作成後、BobはREADME.mdを作成します。
![alt text](assets/images/image-29.png)

**注意:** 生成AIの生成結果によってREADME.mdの作成のタイミングは変わることがあります。

## サイトのテスト

### 1. 作成したサイトを開く

Bobが "open ~.html" というコマンドの実行を提案してくるので、コマンドを実行し、サイトを開き、作成されたサイトを確認します。
![alt text](assets/images/image-28.png)

**注意:** Bobからの提案がない場合は、Bobに「作成したサイトをブラウザで確認したい」と伝えてください。

UIの確認をし、予定機能が動くかを確認してみてください。

**確認項目:**

#### サイトの動き方

#### 実装する機能

- カテゴリ絞り込み
- 検索窓でキーワード検索
- 質問をクリックすると答えが開く（アコーディオン）
