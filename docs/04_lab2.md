# LAB2 - 機能開発
![LAB2 Header](assets/images/04_lab2/lab2_header.png)

Lab1で作成したサイトをさらにブラッシュアップし、4つの機能を新たに加えます。

**注意:** LAB1ですでに追加済みの機能はスキップしてください。
![Additional Features](assets/images/04_lab2/additional_features.png)

- **Func1: Bob画像素材の挿入** ⭐ - Bob.svgをFAQサイト左上に挿入
- **Func2: Bob製品サイトURLの埋め込み** ⭐ - 画像クリックで製品サイト（https://www.ibm.com/products/bob）に遷移
- **Func3: テーマカラーの変更** ⭐⭐ - Bobのイメージカラー（青〜紫のグラデーション）に変更
- **Func4: ダークモードテーマの追加** ⭐⭐⭐ - ライト/ダークモード切り替えボタンを追加


## ハンズオン操作

### セッションの切り替え

「新しいタスクを開始」を押して、セッションを切り替えてください。
![New Task Button](assets/images/04_lab2/new_task_button.png)

**確認:**
- LAB1の履歴がサイドバーに表示されます
- 新しいチャット画面が開きます

### Promptの入力

**Codeモード**を選択し、今回の機能追加のためのPromptを追加します。

**Promptの例:**

```
作成したFAQサイトに以下の機能をつけてください
1.サイト左上にBobの画像の挿入（ @/bob.svg ）
2.1の画像にリンクを挿入（https://www.ibm.com/products/bob ）
3.サイトテーマカラーをBob画像のカラーに変更( @/bob.svg )
4.ダークテーマの追加
```


### 実装計画の立案

#### 1. ファイルの読み込み
BobがFAQサイトの既存ファイルを読み込みます。
![File Reading](assets/images/04_lab2/file_reading.png)

#### 2. Todo Listの作成

Bobが実装計画をTodo List形式で提示します。
![TODO List Creation](assets/images/04_lab2/todo_list_creation.png)


**ユーザーのアクション:**
- Todo Listの内容を確認
- 「承認」をクリック

### コードの修正

Bobが順番にコードを修正していくため、その進行状況を確認します。


### サイトの確認

#### 1. サイトの確認

コード修正のタスクが完了したのち、Bobがサイトを開くコマンドを実行します。

![alt text](assets/images/image-6.png)

#### 2. サイトの機能の確認

作成されたサイトに、追加した機能が正しく反映されているか確認してください。

**確認項目:**

#### Func1: Bob画像の挿入
- [ ] Bob.svgが左上に表示されている
- [ ] 適切なサイズで表示されている
- [ ] レイアウトが崩れていない

#### Func2: 製品サイトへのリンク
- [ ] 画像をクリックできる
- [ ] 新しいタブで製品サイトが開く
- [ ] ホバー時にカーソルが変わる

#### Func3: テーマカラーの変更
- [ ] Bobのイメージカラーが適用されている
- [ ] 全体的に統一感がある
- [ ] 視認性が保たれている

#### Func4: ダークモード
- [ ] 切り替えボタンが表示されている
- [ ] ボタンをクリックでモードが切り替わる
- [ ] 設定が保存される（ページをリロードしても維持）
- [ ] すべての要素が適切に表示される

### 3. エラーの対処

**注意:** サイトがエラーになっている場合は、近くのIBM社員に相談のうえ、Bobにも状況を伝えてください。

**よくあるエラー:**
- 画像パスの問題
- CSSの競合
- JavaScriptのエラー
- ダークモードの不完全な実装

**対処方法:**
1. エラーメッセージを確認
2. Bobにエラー内容を伝える
3. Bobの修正提案を確認
4. 修正を適用
5. 再度テスト
